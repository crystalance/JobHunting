Google  Google

流程：OA（NG 才有）→ R1（Coding + BQ，BQ 可能在前也可能在后，约 45 min）→ R2 / Onsite（2~3 轮 Coding，部分含 OOD）→ HC → Team Match → Offer。

沟通：clarify、think out loud、复杂度分析、dry run；面试官多次明示「efficiency 不是关键」。

BQ 高频清单：

  • Why Google
  • tight deadline / too much work
  • a failure / made mistake
  • 不同背景 / 工作风格的协作（WFH、跨地区）
  • 项目优先级频繁变更
  • 你 lead 的有业务影响的技术决策
  • good characteristics of a manager
  • initiate a project / feature
  • 修改 initial proposal 的经历
  • 在 low culture / 队友不努力的环境怎么改进
  • 假如你是 senior director / 更大权力会怎么做

Amazon  Amazon

流程：OA（HackerRank，2 题 medium，强烈建议刷真题，原题命中率高；NG 的「大 OA」2.5h 含 AI Coding + Work Simulation）→ VO 2~3 轮 back-to-back。

每轮格式：2 BQ + 3 follow-up + 1 Coding（部分组改为 OOD）。

Coding 难度：低于 LC，重点在 clarify、edge case、trade-off。常见原题：幂集 / DFS 变体 / Compare Version Number / KNN / 合并区间。

OOD 高频：Product Event Router（新难题）、餐厅/系统设计类。

BQ 必须对齐 LP（Leadership Principles），准备 6-8 个 STAR 故事：tight deadline、harsh feedback、dive deep、bad decision、conflict、get out of comfort zone、proudest project、GenAI 使用 + verify + risk control（2026 新增重点）。

共同注意事项

  • 别 syntax 卡壳；
  • 写代码时多注释 + 边写边讲；
  • 写完务必 dry run（避免 dead code、逻辑 bug）；
  • BQ 答案补足细节，否则会被连环追问；
  • 不会就大白话 clarify，最忌张冠李戴的「背题感」。
