The v3-reward redesign (drop the collapse-causing turn-cost, keep coverage weight 0.8, add a hard −0.10 no-table floor, KL 0.005, init from v3-SFT) delivered the combined win: GRPO-v3 is more accurate AND cheaper/faster than the v3-SFT base — with no collapse (the exact failure mode of the earlier v2-reward). This is the model to keep.

Headline: ~−14–16% cost / query and −9% latency, at +9% higher EN accuracy.

Why latency (−9%) < reasoning-call cut (−16%): episode wall-clock is gated by the slow search I/O (Bing + gpt-4.1-mini summarize), which only dropped ~9% (search calls 4.4→4.0). Latency tracks search, not reasoning turns. Caveat: latency was measured with both models running in parallel sharing Azure rate limits — real but contention-influenced; the call-count cuts (−14/−16%) are clean and contention-free.

4. Training stability (recap)

21 steps, ~5h, no collapse: final entropy 0.545, grad_norm 0.60, response length stable ~12.7k, constant_fill = 0.0, and the hard −0.10 no-table floor active every step. Classic unlearn-then-relearn (dipped steps 3–9, recovered from step 12). Contrast the earlier v2-reward, which collapsed at step ~10 (turn-cost → stop-early → no-table absorbing state). Full curve in the 7/21 doc.

5. What made it work

  • Better SFT data first (f1_row-gated v3 traces) put coverage into the sample distribution → RL had something good to amplify (pass@k→pass@1; here pass@4 also rose, so the ceiling lifted too).
  • Removed the turn-cost term — it was the root cause of the v2 collapse. Efficiency was NOT forced; it came from KL-anchoring to the already-efficient v3 policy + reinforcing outcome. Result: efficiency improved anyway (−16% turns) without the collapse.
  • Hard no-table floor + coverage weight 0.8 kept "finish" strictly > "give up" and pushed row coverage.
