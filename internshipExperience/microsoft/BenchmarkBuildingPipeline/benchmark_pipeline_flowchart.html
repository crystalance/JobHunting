Custom Benchmark — Benchmark-Generating Pipeline

A web-search benchmark builder for Excel-Copilot-style tasks. Raw user query + workbook dump → structured fact_spec → web-searched, independently-verified fact_spec_gold → freshness maintenance → scoring of any candidate agent. Source: custom_benchmark/

Isolated session re-fetches the cited URL independently → verified: bool. Routes by source type ↓ — the built-in fetch can't read binaries and won't run JS, so each type gets its own extractor.

L4 · Verifier — source-type routing  each path pre-parses evidence, then a Copilot session judges verified with a path-specific system prompt

① Normal HTML AGENT

② Binary file CODE

.xlsx · .xls · .csv · .pdf — built-in fetch returns null on binaries. We download & parse rows matching the key/value, inject ≤14k chars of text into the prompt.

③ JS-rendered / SPA CODE

Opt-in GT_RENDER_FETCH. Value injected by client JS, absent from static HTML. Headless Chromium (Playwright) renders, waits for network idle, extracts innerText lines near the key.

◇ Route by tier CODE

For each instance, resolve a skill by the evidence_url host. No skill → unroutable. Picks Tier A vs Tier B.

refresh_agent.run_instance() · registry.resolve_by_host()

Ⓐ Tier A — replay CODE

Deterministic, offline. Re-run the parser on the frozen build-time snippet snapshot — proves the producer reproduces the product, zero network flakiness.

parser(snapshot, key, type)

Ⓐ Tier A — live AGENT

Best-effort --live. Call the skill's fetch() against the live source to pull a fresh value from the same authority.

module.fetch({key, source_url})

Ⓑ Tier B — live playbook AGENT

Anti-scrape sources (no clean parser). Build query from identity → web_search → LLM playbook extraction. Deferred in replay (no snapshot).

tier_b.run_playbook(skill, key, type, web_search, llm)

Design notes. Spec primitives are shared while each case's rubric differs (Mind2Web-2 critical-node gating × WideSearch cell-level F1). Filling and verification are deliberately asymmetric — separate Copilot sessions. Tier A (deterministic, replayable from frozen snapshots) vs Tier B (LLM playbook, anti-scrape) keep dynamic ground truth maintainable. Generated from a static read of custom_benchmark/ on 2026-06-18.
